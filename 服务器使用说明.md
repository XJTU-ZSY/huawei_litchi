# Python 服务端使用说明

## 启动

```bash
python server.py --host 0.0.0.0 --port 8081
```

常用参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--host` | `0.0.0.0` | 监听地址 |
| `--port` | `8081` | TCP 端口 |
| `--tick-ms` | `500` | 每个结算帧等待客户端 `action` 的毫秒数 |
| `--duration-round` | `600` | 最大结算帧 |
| `--seed` | `20260630` | 本地确定性种子 |
| `--log-dir` | `logs` | 日志目录 |
| `--allowed-player-ids` | `1001,1002` | 允许注册的 playerId，逗号分隔；传空字符串表示允许任意两个 |
| `--verbose` | 关闭 | 控制台输出调试日志 |

客户端按任务书格式连接：

```bash
./start.sh <playerId> <host> <port>
```

服务端会等待两名客户端发送 `registration`，随后下发 `start`；双方 `ready` 后开始循环下发 `inquire` 并接收 `action`。

## 协议

TCP 帧格式为：

```text
5 位十进制 UTF-8 body 长度 + JSON body
```

服务端支持的上行消息：

| 消息 | 说明 |
| --- | --- |
| `registration` | 注册 `playerId`、`playerName`、`version` |
| `ready` | 客户端处理完 `start` 后确认 |
| `action` | 按当前 `inquire.round` 提交动作 |

服务端下行消息：

| 消息 | 说明 |
| --- | --- |
| `start` | 对局、地图、资源、任务模板和玩家阵营 |
| `inquire` | 当前公开状态、上一帧事件和动作结果 |
| `over` | 结算结果 |
| `error` | 协议层错误或当前包未进入规则结算 |

## 日志

日志默认写入 `logs/`：

| 文件 | 内容 |
| --- | --- |
| `server_YYYYMMDD_HHMMSS.log` | 控制台同款可读日志，包含连接、注册、每帧摘要、结束结果 |
| `<matchId>.audit.jsonl` | 结构化审计日志，每行一个 JSON 记录 |
| `<matchId>.replay.jsonl` | 复盘日志，只包含服务端下发的 `start/inquire/over/error` 消息 |

审计日志重点记录：

| `kind` | 内容 |
| --- | --- |
| `message_in` | 客户端上行消息完整 JSON 内容 |
| `message_out` | 服务端下行消息完整 JSON 内容 |
| `action_received` | 客户端动作数组 |
| `round_actions_closed` | 本帧收到/缺失动作的玩家 |
| `round_settled` | 本帧事件、动作判定、玩家状态、节点状态 |
| `protocol_error` | 立即返回的 `error` 详情 |
| `over` | 最终计分和胜负 |

正常客户端的复盘工具建议使用 `<matchId>.replay.jsonl`，它比审计日志更干净，没有客户端上行消息和裁判内部记录。

## 测试

```bash
python -B -m unittest -v
```

`-B` 用于避免在部分 Windows 目录权限下写 `__pycache__`。
